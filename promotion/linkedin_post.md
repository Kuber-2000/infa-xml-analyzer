**I built an open-source CLI tool for data engineers working with Informatica PowerCenter.**

If you've ever had to dig through massive Informatica XML exports to find which mappings live in which folders, trace data lineage, or just get a count of mapplets -- you know the pain.

**infa-xml-analyzer** solves this in one command:

- Parse Informatica PowerCenter XML exports instantly
- Extract folders, mappings, mapplets, sources, targets, and transformations
- Trace full data lineage (source -> transformations -> target)
- Get clean terminal tables or export to CSV for further analysis
- Scan entire directories of XML files at once

Built with Python, lxml, pandas, and Click. MIT licensed.

Whether you're doing ETL documentation, migration planning, or impact analysis -- this tool gives you visibility into your Informatica metadata without expensive commercial software.

Check it out: https://github.com/Kuber-2000/infa-xml-analyzer

Would love feedback from anyone working with Informatica or ETL tooling.
